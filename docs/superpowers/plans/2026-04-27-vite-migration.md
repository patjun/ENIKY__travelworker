# Vite Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the unused Laravel Mix build pipeline with Vite + Tailwind, register a Filament custom theme that consumes shared branding tokens (primary `#5cd0dd`, accent `#39527d`, Inter font, Tailwind default radius), and reduce npm vulnerabilities to zero — without breaking the 366-test suite.

**Architecture:** Single Vite config with three inputs (frontend CSS, frontend JS, Filament theme CSS). One root `tailwind.config.js` defines the branding tokens; the Filament theme's `tailwind.config.js` re-exports those tokens to keep a single source of truth. `welcome.blade.php` switches from inline styles to `@vite()`. Filament's color API is mapped to the brand primary.

**Tech Stack:** Vite 5, `laravel-vite-plugin` 1, Tailwind 3, PostCSS 8, Autoprefixer 10, `@fontsource/inter` 5, axios 1, Filament v3 custom theme.

**Spec:** [docs/superpowers/specs/2026-04-27-vite-migration-design.md](../specs/2026-04-27-vite-migration-design.md)
